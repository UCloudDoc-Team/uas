# 编辑生命周期挂钩

## 操作场景

您可以修改已经创建的生命周期挂钩配置。

修改成功后，新配置会立即保存到伸缩组中，但**只影响后续新触发的生命周期挂钩活动**。

已经创建的生命周期挂钩活动不会受影响。也就是说，如果某台主机已经处于“加入挂起中”或“移出挂起中”，即使此时修改或禁用对应生命周期挂钩，该主机当前的挂钩活动仍会按照创建活动时的配置继续执行，直到手动完成或超时。

## 操作步骤

1. 登录弹性伸缩控制台。
2. 进入目标伸缩组详情页。
3. 选择“生命周期挂钩”页签。
4. 找到需要修改的生命周期挂钩。
5. 单击该生命周期挂钩右侧的“编辑”。
6. 参考“创建生命周期挂钩”中的参数说明，修改配置。
7. 单击“确定”，完成修改。

## 可修改参数

生命周期挂钩创建后，**不支持修改名称**。

通常可以修改以下配置：

| 参数 | 说明 |
|---|---|
| 伸缩活动类型 | 修改挂钩触发的活动类型，例如扩容或缩容 |
| 超时时间 | 修改后续新挂钩活动的等待时长 |
| 执行策略 | 修改挂钩超时或手动结束后的默认执行结果 |
| 通知方式 | 修改是否发送 **USNS** 通知 |
| 通知主题 | 修改 **USNS** 通知主题 |
| 启用状态 | 修改生命周期挂钩是否启用 |

## 生效范围

生命周期挂钩的修改采用**活动快照**机制。

当生命周期挂钩被触发时，系统会创建一条生命周期挂钩活动，并将当时的配置保存到活动中。后续即使修改生命周期挂钩，也不会改变已经创建的活动。

| 场景 | 是否受新配置影响 |
|---|---|
| 修改前已经进入挂起中的主机 | **不受影响**，继续按照原活动配置执行 |
| 修改后新触发的扩容主机 | **受新配置影响** |
| 修改后新触发的缩容主机 | **受新配置影响** |
| 已经创建但未完成的生命周期挂钩活动 | **不受影响** |
| 后续新创建的生命周期挂钩活动 | **受影响** |

## 操作结果

编辑完成后：

- 生命周期挂钩配置更新成功。
- 新配置会用于后续新创建的生命周期挂钩活动。
- 已经处于**“加入挂起中”或“移出挂起中”的主机不会跳过当前挂钩，也不会重新计算超时时间。
- 如果修改了通知配置，只有后续新触发的挂钩活动会使用新的通知配置。
- 如果禁用了生命周期挂钩，后续新伸缩活动不会再创建该挂钩活动；但已有挂钩活动仍会继续执行。

## 注意事项

- 生命周期挂钩名称创建后不支持修改。
- 修改超时时间不会影响已经创建的挂钩活动。
- 修改执行策略不会影响已经创建的挂钩活动。
- 禁用生命周期挂钩不会结束当前正在挂起的主机。
- 如果需要结束当前挂起中的主机，请在实例管理中手动结束对应生命周期挂钩活动。
