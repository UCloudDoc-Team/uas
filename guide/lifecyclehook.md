# 生命周期挂钩概述
生命周期挂钩用于控制伸缩组中主机的扩容和缩容流程。
当主机扩容完成或即将被删除时，生命周期挂钩可以暂时挂起主机，让业务在指定时间内完成初始化、数据备份、请求摘流等操作。挂钩完成或超时后，系统根据执行结果继续处理主机。
## 适用场景
### 扩容场景
主机创建完成后，需要先执行一些初始化操作，再正式投入使用，例如：
- 安装或启动业务程序
- 下载配置文件
- 注册服务
- 绑定辅助网卡
- 执行业务初始化脚本

### 缩容场景
主机停止或删除前，需要先完成业务收尾操作，例如：
- 从服务发现中注销
- 等待存量请求处理完成
- 备份数据
- 上传日志
- 执行下线脚本

## 挂钩类型
目前支持以下两种生命周期挂钩：

| 挂钩类型  | 触发时机                     | 主机等待状态 |
|-----------|------------------------------|--------------|
| SCALE_OUT | 主机创建完成并进入运行状态后 | 加入挂起中   |
| SCALE_IN  | 主机被标记删除、停止或销毁前  | 移出挂起中   |

没有配置对应类型的生命周期挂钩时，主机会直接进入下一阶段。

## 扩容流程
当云主机已经创建完成，并且存在启用的 **`SCALE_OUT`** 挂钩时：

1.主机进入 **`PostCreate`** 状态。

2.系统为每个挂钩创建一条生命周期活动。

3.生命周期活动进入 Pending 状态。

4.系统发送挂钩开始事件和通知。

5.等待业务主动完成挂钩，或者等待挂钩超时。

6.所有扩容挂钩结束后，根据执行结果处理主机。

执行结果：
- 所有挂钩均为 **`CONTINUE`** ：主机进入 **`Running`** 。
- 任意挂钩为 **`REJECT`** ：本次扩容被拒绝，主机进入删除流程。
![扩容流程](/images/lifecyclehook.png)

## 缩容流程
存在启用的 **`SCALE_IN`** 挂钩时：

1.主机被标记为删除。

2.主机进入 **`PreStop`** 状态。

3.系统为每个缩容挂钩创建生命周期活动。

4.系统发送挂钩开始事件和通知。

5.等待所有挂钩主动完成或超时。

6.挂钩全部结束后，继续停止和销毁主机。

当前实现中，缩容挂钩的 **`CONTINUE`** 和 **`REJECT`** 都会继续删除主机。 **`REJECT`** 不会让主机回滚到 **`Running`** 。
![缩容流程](/images/lifecyclehook2.png)

## USNS 通知说明

创建生命周期挂钩时，如果“通知方式”选择 **`USNS`** ，系统会在生命周期挂钩触发时发送通知。

通知内容用于告知用户当前有主机进入挂起状态，用户可以根据通知中的信息执行自定义操作，并在操作完成后手动结束挂起。

### 通知触发时机

| 场景     | 通知时机                                     |
| -------- | -------------------------------------------- |
| 扩容活动 | 主机创建完成并进入 **“加入挂起中”** 时发送通知 |
| 缩容活动 | 主机进入 **“移出挂起中”** 时发送通知           |

### 通知内容

USNS 通知中会包含生命周期挂钩活动相关信息，通常包括：

| 字段                | 说明                              |
| ------------------- | --------------------------------- |
| **伸缩组 ID**       | 当前触发生命周期挂钩的伸缩组      |
| **主机 ID**         | 当前进入挂起状态的主机            |
| **生命周期挂钩 ID** | 触发的生命周期挂钩                |
| **生命周期活动 ID** | 本次挂钩活动 ID，用于手动完成挂起 |
| **伸缩活动类型**    | **`SCALE_OUT`** 或 **`SCALE_IN`** |
| **默认执行策略**    | **`CONTINUE`** 或 **`REJECT`**    |
| **超时时间**        | 主机保持挂起状态的最长时间        |

## 回调处理

如果用户需要通过程序自动处理生命周期挂钩，可以在收到 USNS 通知后，根据通知中的生命周期活动 ID 调用完成接口，手动结束挂起。

完成时可以选择：

| 执行结果     | 说明                                             |
| ------------ | ------------------------------------------------ |
| **Continue** | 继续当前扩缩容流程                               |
| **REJECT**   | 拒绝当前扩缩容流程。扩容场景下主机会进入删除流程 |

## 注意事项

- 只有通知方式选择 **`USNS`** 时，才会发送生命周期挂钩通知。
- 如果通知方式选择 **`None`** ，主机仍会进入挂起状态，但不会发送通知。
- **USNS 通知只表示挂钩活动已触发，不代表挂钩已完成。**
- 如果用户未手动完成挂钩，系统会在超时时间到达后自动结束挂起，并按执行策略继续处理。
- **禁用或修改生命周期挂钩不会影响已经创建的 USNS 通知或已触发的挂钩活动。**
- 如果同一阶段触发多个生命周期挂钩，可能会收到多条 USNS 通知；主机需要等待所有对应挂钩活动完成后才会继续流转。
