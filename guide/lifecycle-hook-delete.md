# 删除生命周期挂钩

## 操作场景

您可以删除已经创建的生命周期挂钩。

生命周期挂钩删除后**不可恢复**。删除后，该挂钩不会再对后续新触发的扩容或缩容活动生效。

## 操作说明

当前实现中，生命周期挂钩采用**“活动快照”**机制。

当挂钩被触发时，系统会创建生命周期挂钩活动，并将触发时的挂钩配置保存到活动中。因此：

- 删除生命周期挂钩后，不会再创建新的挂钩活动。
- 已经创建的生命周期挂钩活动仍会继续执行。
- 已经处于**“加入挂起中”**或**“移出挂起中”**的主机，不会因为删除挂钩而立即跳过挂起。
- 当前挂起中的主机仍会等待已有活动手动完成或超时。

## 操作步骤

1. 登录弹性伸缩控制台。
2. 进入目标伸缩组详情页。
3. 选择“生命周期挂钩”页签。
4. 找到需要删除的生命周期挂钩。
5. 单击该生命周期挂钩右侧的“删除”。
6. 在确认弹窗中单击“删除”，完成操作。

## 删除后的影响

| 场景 | 删除后的行为 |
|---|---|
| 后续新触发的扩容活动 | 不再触发该生命周期挂钩 |
| 后续新触发的缩容活动 | 不再触发该生命周期挂钩 |
| 删除前已经创建的挂钩活动 | **继续执行** |
| 已处于“加入挂起中”的主机 | 继续等待已有挂钩活动完成或超时 |
| 已处于“移出挂起中”的主机 | 继续等待已有挂钩活动完成或超时 |
| 删除前已发送的通知 | 不受影响 |

## 多个生命周期挂钩的情况

如果同一伸缩活动触发了多个生命周期挂钩，主机需要等待所有已创建的挂钩活动完成后，才会继续流转。

删除其中一个生命周期挂钩后：

- 不影响已经创建的该挂钩活动。
- 不影响其他已经创建的挂钩活动。
- 后续新伸缩活动不会再创建被删除挂钩对应的活动。

例如，某台主机扩容时已经触发了两个 **`SCALE_OUT`** 挂钩：

```text
hook-a
hook-b
```

此时删除 **`hook-a`**：

- 当前主机仍然需要等待 **`hook-a`** 和 **`hook-b`** 对应的已有活动完成；
- 后续新扩容主机只会根据剩余挂钩配置创建活动。

## 操作结果

删除成功后：

- 生命周期挂钩列表中不再显示该挂钩。
- 后续扩容或缩容不会再触发该挂钩。
- 已有挂钩活动继续按照创建时的配置执行。
- 如果当前没有其他未完成挂钩活动，主机会在已有活动完成或超时后继续进入下一阶段。

## 注意事项

- 删除生命周期挂钩**不可撤销**。
- 删除生命周期挂钩不会自动结束当前挂起中的主机。
- 删除生命周期挂钩不会修改已经创建的生命周期挂钩活动。
- 如果需要立即结束当前挂起状态，请在“实例管理”中手动结束对应挂钩活动。
- 如果删除的是最后一个挂钩，后续同类型伸缩活动将不会再进入该挂起流程。
