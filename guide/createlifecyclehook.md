# 创建生命周期挂钩

## 操作场景
生命周期挂钩用于在伸缩组执行扩容或缩容时，将主机临时挂起，给用户预留一段时间执行自定义操作。

## 使用限制
- 一个伸缩组可以配置多个生命周期挂钩，最多10个。
- 生命周期挂钩支持两种伸缩活动类型：扩容活动 **SCALE_OUT**、缩容活动 **SCALE_IN**
- 生命周期挂钩超时时间建议设置为业务自定义操作所需的最大时间。
- 如果同一阶段存在多个生命周期挂钩，主机需要等待所有对应挂钩活动完成后，才会继续流转。
- 禁用生命周期挂钩后，不会影响已经处于挂起中的主机；只会影响后续新创建的生命周期挂钩活动。

## 操作步骤

1.登录弹性伸缩控制台。

2.进入目标伸缩组详情页。

3.选择“生命周期挂钩”页签。

4.单击“新建生命周期挂钩”。

5.配置生命周期挂钩参数。


| 参数         | 说明                                                                          | 示例           |
|--------------|-------------------------------------------------------------------------------|----------------|
| 名称         | 生命周期挂钩名称。用于识别该挂钩。                                              | scale-out-init |
| 伸缩活动类型 | 指定挂钩在扩容还是缩容时触发。扩容对应 **SCALE_OUT**，缩容对应 **SCALE_IN**。            | SCALE_OUT      |
| 超时时间     | 主机进入挂起状态后最长等待时间。超时后系统会自动结束挂起，并按执行策略继续处理。 | 600s           |
| 执行策略     | 挂钩结束后的默认处理方式。支持 **CONTINUE** 和 **REJECT**。                             | CONTINUE       |
| 通知方式     | 生命周期挂钩触发时是否发送通知。当前支持 **USNS** 或 **None**。                         | USNS           |

## 操作结果
生命周期挂钩创建后：
- 后续扩容主机在创建完成后会进入“加入挂起中”
- 后续缩容主机在删除前会进入“移出挂起中”
- 控制台可以看到主机的生命周期状态和剩余等待时间
- 如果配置了 USNS，生命周期挂钩开始时会发送通知
- 挂钩完成或超时后，主机会继续进入下一阶段

## 注意事项
- 如果主机已经进入挂起状态，禁用生命周期挂钩不会跳过当前挂钩活动
- 扩容场景下，如果执行结果为 **REJECT**，主机会进入删除流程
- 缩容场景下，**REJECT** 不会恢复主机到服务中
- 删除伸缩组时，系统会先删除主机，并触发缩容生命周期挂钩
